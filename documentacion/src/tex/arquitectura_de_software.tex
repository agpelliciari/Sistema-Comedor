\section{Arquitectura de Software}

Se decidió utilizar un patrón de arquitectura enterprise, vinculado con el patrón por capas. Se definieron las siguientes capas:

\node[box, dashed] (presentacion) {Capa de presentación \\ \textit{(Front End)}};
\node[box, below=of presentacion] (controlador) {Capa de controlador \\ \textit{(API REST)}};
\node[box, below=of controlador] (servicio) {Capa de servicio \\ \textit{(Lógica de negocio y dominio)}};

\end{center}

\begin{itemize}
  \item \textbf{Capa de presentación}: encargada de presentar la información a los distintos usuarios y permitir, a partir de eventos generados por el usuario, interactuar con la capa de controlador. Siguiendo la arquitectura enterprise, esta capa brinda vistas distintas dependiendo del tipo de usuario que accede a la aplicación (estudiante/cliente, personal de cocina/caja y encargado/administrador).

  \item \textbf{Capa de controlador}: expone una API REST que recibe las solicitudes de la capa de presentación, realiza el mapeo entre las llamadas HTTP y los casos de uso del sistema, y delega la lógica de negocio en la capa de servicio y negocio. También es responsable de aplicar las políticas de seguridad de la API (autenticación y autorización por rol).

  \item \textbf{Capa de servicio y negocio}: implementa las funcionalidades necesarias para los distintos casos de uso, orquestando las reglas de negocio del dominio (pedidos, menú, descuentos, usuarios, etc.). Esta capa encapsula la lógica de aplicación y utiliza la capa de persistencia para acceder a los datos.

  \item \textbf{Capa de persistencia}: ofrece a la capa de servicio y negocio el servicio de persistir, almacenar y recuperar la información, a través de repositorios que abstraen el acceso a la base de datos relacional.
\end{itemize}
